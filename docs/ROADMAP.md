## 1. 비전 & 포지셔닝

### 핵심 비전

> "개발자의 AI 활용 역량을 측정하고 증명하는 대표 도구"

이력서는 무엇을 만들었는지를 말한다. Autopolio는 **어떻게 만들었는지**를 말한다.
AI 도구를 얼마나 깊이, 얼마나 전략적으로 활용했는지를 수치화하고, 포트폴리오로 증명한다.

### 왜 지금인가

- AI 코딩 도구(Claude Code, Copilot, Cursor)가 주류가 되면서 "AI 활용 능력" 자체가 채용 기준이 되고 있다.
- 기존 이력서 플랫폼은 결과물(프로젝트, 기술스택)만 보여준다. 과정과 방식은 보여주지 못한다.
- GitHub 커밋 내역, CLI 사용 로그, 코드 품질 변화 추이는 AI 활용 역량의 객관적 증거다.
- 오픈소스 + 로컬 우선(Electron) 구조는 민감한 코드 분석에 대한 신뢰 장벽을 낮춘다.

### 경쟁 제품 대비 차별화

| 구분 | LinkedIn Resume Builder | resume.io / Novoresume | Autopolio |
|------|------------------------|----------------------|-----------|
| 데이터 소스 | 수동 입력 | 수동 입력 | GitHub 자동 분석 |
| AI 활용 측정 | 없음 | 없음 | **핵심 기능** |
| 코드 증거 연결 | 없음 | 없음 | 커밋/PR/코드 품질 |
| 로컬 실행 | 없음 | 없음 | Electron 지원 |
| 오픈소스 | 없음 | 없음 | 완전 공개 |
| 커스텀 템플릿 | 제한적 | 유료 | 무제한 |
| 한국 플랫폼 지원 | 없음 | 없음 | 사람인/원티드/리멤버/점핏 |

## 2. 단기 로드맵 (v0.2 ~ v0.4)

현재 v0.1.0의 안정성을 유지하면서 "AI 활용 측정"이라는 핵심 포지셔닝을 위한 기반을 구축한다.

### v0.2 — AI 활용 지표 기반 구축

**목표**: AI 도구 사용 패턴을 커밋 데이터에서 자동 탐지한다.

- **[P0] AI 커밋 패턴 감지기**
  - 커밋 메시지에서 AI 도구 관련 패턴 자동 추출
  - 탐지 대상: `Co-authored-by: GitHub Copilot`, `claude`, `cursor`, `copilot`, `gpt` 키워드
  - 탐지 결과를 `contributor_analyses` 테이블에 저장
  - AI 보조 커밋 비율, 주요 사용 도구, 도구별 기여 영역(frontend/backend/test) 산출

- **[P0] AI 활용 지표 대시보드 (기초)**
  - Dashboard 페이지에 "AI 활용 현황" 섹션 추가
  - 주요 지표: AI 보조 커밋 수, AI 활용률(%), 자주 사용한 AI 도구 Top 3
  - 프로젝트별 AI 활용도 비교 차트

- **[P1] `.claude/` 디렉토리 분석**
  - 레포 내 `CLAUDE.md`, `.claude/commands/`, `.claude/settings.json` 존재 여부 탐지
  - Claude Code 커스텀 명령어 수, 프로젝트별 AI 설정 복잡도 측정
  - "AI 워크플로우 성숙도" 점수 산출 (0~100)

- **[P1] 포트폴리오 공개 페이지 (MVP)**
  - 사용자별 공개 URL 생성 (`/portfolio/{username}` 또는 단축 링크)
  - 공개 범위 설정: 전체 공개 / 링크 공유 / 비공개
  - 기본 포트폴리오 카드 (프로젝트 목록, 기술스택, AI 활용 지표)
  - 소셜 공유용 OG 메타태그

- **[P2] 글로벌 플랫폼 템플릿 추가**
  - LinkedIn 프로필 섹션 포맷 (About, Experience, Skills)
  - Indeed 이력서 포맷
  - 영문 이력서 기본 템플릿 (ATS 친화적 단일 컬럼)

### v0.3 — 분석 품질 심화

**목표**: 단순 커밋 통계를 넘어 코드 품질과 성장 궤적을 보여준다.

- **[P0] 코드 복잡도 추이 분석**
  - 주요 파일의 라인 수, 함수 수 시계열 변화 추적
  - 리팩토링 커밋 감지 (라인 감소 + 테스트 추가 패턴)
  - "코드 개선 기여도" 지표 — 단순 추가가 아닌 정리/개선 작업 비중

- **[P0] 테스트 커버리지 패턴 분석**
  - 테스트 파일 비율 시계열 (프로젝트 초기 vs 현재)
  - `feat:` 커밋 대비 `test:` 커밋 비율
  - CI 설정 파일(`.github/workflows/`) 존재 여부 및 복잡도

- **[P1] PR 리뷰 패턴 분석** (GitHub API 확장)
  - PR 평균 리뷰 소요 시간
  - 리뷰 코멘트 수 (주고받은 것 모두)
  - 자동화된 체크(CI, lint, security scan) 통과율

- **[P1] 분석 결과 내보내기 확장**
  - AI 활용 지표를 포함한 성과 보고서 (.md, .docx)
  - LinkedIn 포스팅용 요약 텍스트 자동 생성

- **[P2] AngelList / Wellfound 템플릿 추가**

### v0.4 — 공개 포트폴리오 완성

**목표**: 공유 가능한 포트폴리오 페이지를 완성하고 SEO를 갖춘다.

- **[P0] 포트폴리오 공개 페이지 완성**
  - 커스터마이징 가능한 섹션 순서 및 표시/숨김 설정
  - 테마 선택 (라이트/다크/개발자 스타일)
  - AI 활용 지표 시각화 (레이더 차트, 활용률 배지)
  - 프로젝트별 상세 페이지 (README 스타일)

- **[P0] AI 활용 배지 시스템**
  - shields.io 스타일 배지 생성 (`AI 활용률 78%`, `Claude Code 사용`)
  - GitHub README 삽입용 마크다운 코드 자동 생성
  - 공개 포트폴리오 URL 기반 동적 배지 (실시간 갱신)

- **[P1] 실시간 GitHub 동기화**
  - GitHub webhook 수신 또는 주기적 폴링 (설정 가능)
  - 새 커밋 감지 시 분석 자동 업데이트
  - 포트폴리오 페이지 자동 반영

- **[P2] 다국어 포트폴리오 페이지** (한국어/영어 전환)

## 3. 중기 로드맵 (v0.5 ~ v1.0)

### v0.5 ~ v0.6 — AI Agent 로그 통합

**목표**: Claude Code, Cursor 등 AI 에이전트의 실제 사용 로그를 분석한다.

- **[P0] Claude Code 사용 로그 분석**
  - `.claude/` 디렉토리 구조 분석 (대화 수, 커스텀 명령어, 에이전트 설정)
  - 분석 항목: 사용 빈도, 주요 작업 유형(코드 생성/리팩토링/디버깅/문서화), 프로젝트별 의존도
  - 프라이버시 우선: 모든 분석은 로컬(Electron)에서만 수행, 서버 전송 없음

- **[P0] Cursor / Copilot 활용 패턴 추정**
  - 코드 생성 패턴 추정: 짧은 시간 내 대량 코드 추가 커밋
  - AI 제안 수락 패턴: 표준 패턴 코드 블록 비율 분석
  - 도구별 "추정 활용도" 점수 (실제 로그 없을 때 휴리스틱)

- **[P1] 팀 기여도 시각화**
  - 레포지토리 내 전체 기여자 대비 사용자 기여 비율
  - 팀 내 AI 활용 선도자 여부 지표
  - 코드 리뷰 기여도 (리뷰어로서의 활동)

- **[P1] 성장 궤적 시각화**
  - 6개월/1년 단위 역량 변화 히트맵
  - AI 도구 도입 전후 생산성 변화 (커밋 패턴, 코드 품질 지표)
  - "개발자 성장 타임라인" 인포그래픽 생성

### v0.7 ~ v1.0 — 플랫폼 확장

- **[P0] 플러그인 시스템**
  - 커스텀 분석기 플러그인 API (Python)
  - 커스텀 템플릿 렌더러 플러그인 (JavaScript)
  - npm/PyPI 기반 플러그인 배포 지원

- **[P1] 커뮤니티 템플릿 마켓플레이스 (베타)**
  - 사용자 제작 템플릿 공유 플랫폼
  - 별점/다운로드 수 기반 랭킹
  - GitHub 레포지토리 기반 템플릿 배포

- **[P1] API 공개 (Read-only)**
  - 사용자 포트폴리오 데이터 조회 API
  - 외부 서비스 연동 (Notion, Obsidian 등)

- **[P2] 모바일 앱 (React Native)**
  - 포트폴리오 열람 및 공유 전용

## 5. 기술적 개선 사항

### 인프라 & 성능

- **[P0] PostgreSQL 전환**
  - SQLite는 단일 사용자/개발용으로 유지
  - Docker Compose에 PostgreSQL 서비스 추가
  - Alembic 마이그레이션으로 스키마 관리
  - 공개 포트폴리오 기능은 PostgreSQL 필수

- **[P0] 웹소켓 실시간 진행률**
  - 현재 폴링 방식을 WebSocket으로 전환
  - 분석 단계별 실시간 스트리밍 (FastAPI WebSocket)
  - 프론트엔드: `useWebSocket` 훅 표준화

- **[P1] Redis 캐싱**
  - GitHub API 응답 캐싱 (TTL: 1시간)
  - 분석 결과 캐싱 (TTL: 24시간, 새 커밋 감지 시 무효화)
  - 공개 포트폴리오 페이지 HTML 캐싱

- **[P1] 분석 작업 큐**
  - 현재 단순 백그라운드 태스크를 Celery + Redis 큐로 전환
  - 병렬 레포 분석 지원 (현재 순차 처리)
  - 작업 우선순위 설정

### 코드 품질 & 테스트

- **[P0] 테스트 커버리지 80%+**
  - 현재 주요 서비스 단위 테스트 부재
  - 우선순위: `analysis_workflow.py`, `cli_llm_service.py`, `technology_detection_service.py`
  - pytest + pytest-cov 커버리지 리포트 CI 강제

- **[P0] API 타입 안정성**
  - 모든 Pydantic 스키마 엄격 모드(`model_config = ConfigDict(strict=True)`)
  - 프론트엔드 API 클라이언트 자동 생성 (openapi-typescript)
  - 런타임 타입 불일치 오류 제거

- **[P1] 모바일 반응형 강화**
  - 현재 PC 중심 레이아웃을 태블릿/모바일 브레이크포인트 추가
  - 공개 포트폴리오 페이지는 모바일 우선 설계
  - Lighthouse 모바일 점수 90+ 목표

- **[P1] 에러 추적 & 모니터링**
  - Sentry 연동 (오픈소스 셀프호스팅 옵션 제공)
  - 구조화된 로깅 (`structlog`)
  - 헬스체크 엔드포인트 표준화 (`/api/health`)

- **[P2] Electron 빌드 최적화**
  - 번들 크기 축소 (현재 Python 번들 포함 시 대용량)
  - 자동 업데이트 (`electron-updater`) 안정화
  - 코드 서명 (macOS Notarization, Windows Authenticode)
